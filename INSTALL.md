# 账号管理助手安装教程

本文说明如何从 GitHub Release 安装账号管理助手 Chrome 扩展，以及如何在本地开发时加载源码目录。

## 安装前准备

- 使用 Chrome、Edge 或其他支持 Chromium 扩展的浏览器。
- 下载发布包后需要先解压，不能直接选择 `.zip` 文件加载。
- 如果浏览器已经安装过旧版本，建议先在扩展管理页点击刷新按钮重新加载，或移除旧版本后重新安装。

## 从 Release 安装

1. 打开项目的 Release 页面：

   ```text
   https://github.com/zzwxxxx/WebAccountAssistant/releases
   ```

2. 选择要安装的版本，例如 `v0.1.0`。

3. 在 Assets 中下载对应的压缩包：

   ```text
   WebAccountAssistant-v0.1.0.zip
   ```

4. 将 zip 解压到一个固定目录。

   示例：

   ```text
   ~/Applications/WebAccountAssistant-v0.1.0
   ```

5. 打开浏览器扩展管理页：

   ```text
   chrome://extensions
   ```

6. 打开右上角的“开发者模式”。

7. 点击“加载已解压的扩展程序”。

8. 选择第 4 步解压出来的目录。

9. 安装完成后，浏览器工具栏会出现“账号管理助手”图标。

## 从源码目录安装

如果你是从 GitHub 克隆仓库或在本地开发，可以直接加载项目根目录。

1. 克隆仓库：

   ```bash
   git clone https://github.com/zzwxxxx/WebAccountAssistant.git
   ```

2. 进入项目目录：

   ```bash
   cd WebAccountAssistant
   ```

3. 打开浏览器扩展管理页：

   ```text
   chrome://extensions
   ```

4. 打开“开发者模式”。

5. 点击“加载已解压的扩展程序”。

6. 选择项目根目录。

## 更新扩展

### Release 包安装的更新方式

1. 下载新版本 Release 包。
2. 解压到新的固定目录，或覆盖原来的扩展目录。
3. 打开：

   ```text
   chrome://extensions
   ```

4. 找到“账号管理助手”。
5. 点击扩展卡片上的刷新按钮。
6. 打开插件弹窗或管理页面，确认版本号已经更新。

### 源码目录安装的更新方式

1. 拉取最新代码：

   ```bash
   git pull
   ```

2. 打开：

   ```text
   chrome://extensions
   ```

3. 点击“账号管理助手”扩展卡片上的刷新按钮。

## 首次使用

1. 点击浏览器工具栏中的“账号管理助手”图标。
2. 点击“管理面板”进入完整配置页面。
3. 按项目、环境、字段和账号维护配置。
4. 打开匹配的登录页面后，通过插件弹窗或页面账号面板选择账号并填充。
5. 首次查看敏感字段或导出明文配置时，按提示创建并验证插件通行密钥。

## 常见问题

### 为什么不能直接加载 zip？

Chrome 的“加载已解压的扩展程序”只能选择目录。请先把 `WebAccountAssistant-vX.Y.Z.zip` 解压，再选择解压后的目录。

### 安装后没有看到插件图标怎么办？

打开浏览器工具栏右侧的扩展菜单，找到“账号管理助手”，将它固定到工具栏。

### 修改或更新后页面没有变化怎么办？

打开 `chrome://extensions`，点击“账号管理助手”卡片上的刷新按钮，然后刷新目标网页。

### 敏感字段为什么需要验证？

账号敏感字段会加密保存在本地。查看敏感字段或导出明文配置时，需要通过插件通行密钥验证，降低误查看和误导出的风险。

## 数据安全提醒

- 不要把真实账号、密码、导出的配置 JSON 或 vault 数据提交到 Git 仓库。
- Release 包只包含扩展程序文件，不包含你的本地账号配置。
- 配置数据保存在当前浏览器本地的 `chrome.storage.local` 中。
- 清理浏览器扩展数据或删除系统通行密钥后，原加密数据可能无法解密。
